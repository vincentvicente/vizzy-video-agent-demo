# Vizzy AI Video Agent — Prototype

A data-driven paid-social video generation agent. Paste a URL → automatically produce a 9:16 vertical, 30–60s ad video.

## Locked architecture decisions (prototype scope)

| Dimension | Decision | Notes |
|---|---|---|
| HITL cadence | **γ** · 1 merged checkpoint before clip gen + final accept | User only clicks 2 buttons |
| Checkpoint transparency | **Level 1** (prototype only) | **Production must move to Level 3 + asymmetric + dual-mode** |
| Memory | **Layer 0** (no memory, fresh every run) | Production upgrades to Layer 2 (brand-level memory) |
| Schema tightness | **B** · role picked from enum, LLM orders + sets duration | Lets the LLM show intelligence across different brands |
| Reference path | **A.1** · user uploads only | PRD item 3 rationale degrades to "user expectation" |
| Model choice | **Seedance 2.0 single model** (prototype simplification) | Production restores the three-layer hard-constraint + preference + LLM fine-tuning |
| Retry mode | **Mode 3 · conditional routing table** | QA fault_type → explicit retry from stage |

## Pipeline architecture

```
URL
 ↓
[Strategist]  fetch URL → brand JSON + storyboard JSON (strict schema)
 ↓
[Reference Selector]  user upload → normalized storage (A.1 path, no LLM call)
 ↓
[Director]  storyboard → Seedance API params (per scene)
 ↓
 ↓   γ Checkpoint: only spend money on the next step once the user approves
 ↓
[Clip Generation]  parallel fal.ai → Seedance 2.0
 ↓
[Editor]  ffmpeg stitching + ElevenLabs VO + caption overlay
 ↓
[QA]  Claude vision frame sampling → fault_type
 ↓
  ├─ pass → final video
  └─ fail → conditional routing table → retry from <stage>
 ↓
Final accept / regen
```

## Tech Stack

- **LLM**: Anthropic Claude (Strategist / Director / QA vision)
- **Video generation**: fal.ai → Seedance 2.0 Pro (image-to-video, with reference control support)
- **Voiceover (VO)**: ElevenLabs
- **Stitching**: ffmpeg-python
- **UI**: Streamlit

## Getting started

```bash
pip install -r requirements.txt
cp .env.example .env  # fill in API keys
streamlit run app.py
```

## Design principles

- **legibility-first**: every stage's I/O is strict JSON, drillable on failure
- **Zero tolerance for the uncontrollable / untraceable**: a hard routing table + a fixed retry budget = the LLM can't go off the rails
- **No visible text in model**: all captions / logos / CTA text are added as overlays in the Editor layer, never generated by the video model (avoids the model spitting out garbled English / spelling errors)

## Limitations (prototype scope)

- **3–6 minutes per run**: clip gen is the physical bottleneck
- **$5–15 cost per run**: mostly fal.ai (5–6 scenes × ~$1)
- **demo-friendly fallback**: data/final/fallback.mp4 is a pre-recorded full run, dropped in manually when the live API fails
